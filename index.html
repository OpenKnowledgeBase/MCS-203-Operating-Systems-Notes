Process Scheduling Algorithms

Key Concepts

Scheduling is the process of selecting which process to execute next when multiple processes are ready.

CPU Scheduling Criteria
  • CPU Utilization: Keep CPU as busy as possible
  • Throughput: # of processes completed per time unit
  • Turnaround Time: Time from submission to completion
  • Waiting Time: Time spent waiting in ready queue
  • Response Time: Time from submission to first response

Algorithm Comparison

Algorithm Description Advantages Disadvantages
FCFS First come, first served Simple, fair Convoy effect, poor for short processes
SJF Shortest job first Minimal average waiting time Starvation possible, hard to predict burst time
Priority Based on priority levels Important processes get priority Starvation of low-priority processes
Round Robin Time slices for each process Fair, good response time High overhead with many context switches
Exam Tip

When drawing Gantt charts, always show time units on the x-axis and clearly label each process segment. Calculate waiting time as (start time - arrival time) for each process.

Page Replacement Algorithms

Basic Concepts

When a page fault occurs and no free frames are available, the OS must select a victim page to replace.

Page Fault Handling Steps
  1. Check page table for reference validity
  2. If invalid → terminate process (segmentation fault)
  3. If valid but not in memory → page fault
  4. Find free frame (or select victim via replacement algo)
  5. Schedule disk operation to load page
  6. Update page table and restart instruction

Replacement Algorithms

FIFO (First-In-First-Out):
Replace the oldest page in memory
Suffers from Belady's Anomaly: More frames → more page faults
LRU (Least Recently Used):
Replace the page not used for longest time
Optimal but expensive to implement perfectly
Memory Trick

For LRU, visualize a stack of plates - the least recently used plate is at the bottom. For FIFO, imagine people in a queue - first in line gets replaced first.

Banker's Algorithm

Deadlock Avoidance

Banker's algorithm ensures the system never enters an unsafe state by carefully allocating resources.

Key Data Structures
  • Available: Vector of available resources
  • Max: Maximum demand of each process
  • Allocation: Currently allocated resources
  • Need: Max - Allocation (remaining needs)

Safety Algorithm Steps

  1. Initialize Work = Available, Finish = false for all
  2. Find process Pᵢ where Finish[i] = false and Needᵢ ≤ Work
  3. If found: Work = Work + Allocationᵢ, Finish[i] = true
  4. Repeat until all Finish[i] = true (safe) or no such Pᵢ exists (unsafe)
Verification Tip

When checking for safe sequence, always verify that Need ≤ Work before allocating. The sequence order matters - different valid orders may exist.

Windows 11 OS Features

Key Components

  • NT Kernel: Hybrid kernel architecture
  • Virtual Memory: Demand paging with clustering
  • Security: Windows Hello, TPM 2.0, Secure Boot
  • File System: NTFS with journaling

Process Management

Windows uses preemptive multitasking with priority-based scheduling:

  • 32 priority levels (0-31)
  • Real-time (16-31) and variable (1-15) classes
  • Quantum-based time slicing
  • Multi-core aware scheduling
